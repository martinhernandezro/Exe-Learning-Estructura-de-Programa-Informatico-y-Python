Estructuras de Control en Python

Estructuras de Control en Python

Las estructuras de control permiten modificar el flujo de ejecución de un programa, haciendo que ciertas instrucciones se ejecuten de manera condicional o repetitiva. En Python, existen cuatro tipos principales de estructuras de control:

-Estructuras condicionales (if, elif, else, match)
-Estructuras de bucle o iteración (for, while)
-Estructuras de control de excepciones (try, except, finally)
-Estructura de selección (match-case)

Estructuras condicionales

La sentencia if
La estructura if evalúa una condición y ejecuta el bloque de código si esta es verdadera.

La sentencia if-else
Permite ejecutar un bloque de código si la condición es verdadera y otro si la condición es falsa.

La sentencia if-elif-else
Cuando hay múltiples condiciones, se puede usar elif (else if).

Estructura de Selección con match-case
A partir de Python 3.10, se introdujo la estructura match-case, similar a switch-case en otros lenguajes. Se usa para evaluar una variable y ejecutar diferentes bloques de código en función de su valor.

Explicación:

-match comando: evalúa la variable comando.
-Cada case representa un posible valor de comando.
-El case _: actúa por defecto, ejecutándose si no coincide ningún caso anterior.

Estructuras de Bucle e Iteración

Las estructuras de bucle permiten repetir un conjunto de instrucciones varias veces.

Bucle while
El bucle while ejecuta un bloque de código mientras la condición sea verdadera.

Bucle for
El bucle for se usa para iterar sobre una secuencia (lista, tupla, cadena, etc.).

Uso de range() en for
Se puede utilizar range() para generar una secuencia un números de veces.

Instrucciones break y continue

-break: Finaliza el bucle antes de que termine.
-continue: Salta a la siguiente iteración sin ejecutar las siguientes líneas del bucle.


 

Estructuras de Control de Excepciones
Python permite manejar errores en tiempo de ejecución con try-except.

 

Explicación:


-Se solicita un índice para acceder a un valor en la lista.
-Se pide un número para dividir ese valor.
-Se manejan distintas excepciones:

-IndexError: Si el usuario ingresa un índice fuera de rango.
-ZeroDivisionError: Si intenta dividir por 0.
-ValueError: Si ingresa un valor no numérico.
-Exception: Captura cualquier otro error no previsto.

-finally asegura que siempre se ejecute un mensaje final.
